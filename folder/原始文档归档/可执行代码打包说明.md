# 可执行代码打包说明

## 1. 课程提交推荐形式

本项目是 Flask + React 前后端分离系统，推荐提交“可运行代码包”，而不是强行编译成单个 `.exe` 文件。

原因：

- 后端需要 SQLite 数据库、上传目录、全文索引目录等运行数据。
- 前端需要 Node/Vite 或构建后的静态资源。
- AIGC 视频生成需要外部接口配置。

因此课程材料中的“可执行代码”建议理解为：包含源码、数据库、配置说明和启动脚本的可运行系统。

## 2. 生成可运行发布包

在项目根目录执行：

```bash
chmod +x scripts/build_release.sh
./scripts/build_release.sh
```

生成结果：

```text
release/trip-runnable/
release/trip-runnable.zip
```

发布包包含：

```text
backend/        后端代码、SQLite数据库、配置样例
frontend/       前端代码
folder/         课程设计文档
deploy/         Gunicorn + Nginx 部署配置
启动说明.md     运行步骤
```

发布包不包含：

```text
frontend/node_modules/
frontend/dist/
backend/__pycache__/
backend/whoosh_index/
backend/.env
```

这些属于依赖、构建产物或本地运行缓存，可以重新生成。

## 3. 发布包运行方式

后端：

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python run.py
```

前端：

```bash
cd frontend
npm install
npm run dev
```

浏览器打开：

```text
http://localhost:3000
```

## 4. 如果必须生成单文件可执行程序

后端可以使用 PyInstaller 打包：

```bash
cd backend
pip install pyinstaller
pyinstaller --onefile run.py --name trip-backend
```

生成：

```text
backend/dist/trip-backend
```

但注意：

- 仍然需要携带 `trip.db`。
- 仍然需要 `uploads/`、`.env` 等运行目录。
- 前端仍需要单独构建为静态资源。
- Flask + React 项目强行做单文件可执行，维护成本较高，不如可运行发布包稳定。

## 5. 前端静态构建

如果需要前端构建产物：

```bash
cd frontend
npm install
npm run build
```

生成：

```text
frontend/dist/
```

生产部署时可由 Nginx 托管 `frontend/dist`。

## 6. 答辩说明

可以这样说：

> 本系统提供可运行发布包，包含后端 Flask 服务、前端 React 项目、SQLite 数据库、部署配置和启动说明。由于系统是 Web 应用，课程要求中的可执行代码采用“可运行系统包”的形式提交；后端也可以用 PyInstaller 进一步打包，但实际部署推荐使用 Gunicorn + Nginx。
