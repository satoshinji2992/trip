# 个性化旅游系统 - 前端开发文档

## 一、项目概述

前端采用 **React 18 + Vite 5 + Ant Design 5 + TailwindCSS 3** 技术栈构建，提供完整的用户界面，包括旅游推荐、路线规划、场所查询、旅游日记、美食推荐、室内导航等功能模块。

## 二、项目结构

```
frontend/
├── index.html                  # 入口HTML
├── package.json                # 依赖管理
├── vite.config.js              # Vite配置（含代理）
├── tailwind.config.js          # TailwindCSS配置
├── postcss.config.js           # PostCSS配置
├── public/                     # 静态资源
└── src/
    ├── main.jsx                # 应用入口（React挂载、路由、国际化）
    ├── App.jsx                 # 根组件（布局、路由配置、用户上下文）
    ├── index.css               # 全局样式（Tailwind指令、地图样式）
    ├── services/
    │   └── api.js              # API请求封装（Axios实例、所有接口定义）
    └── pages/
        ├── HomePage.jsx        # 首页（推荐、搜索入口、功能导航）
        ├── ScenicListPage.jsx  # 景区/校园列表（搜索、过滤、排序）
        ├── ScenicDetailPage.jsx# 景区/校园详情（信息展示、功能入口）
        ├── RoutePlanPage.jsx   # 路线规划（Canvas地图、最短路径、多点规划）
        ├── FacilityPage.jsx    # 场所查询（附近设施、距离排序）
        ├── DiaryListPage.jsx   # 日记列表（浏览、搜索、推荐、全文搜索）
        ├── DiaryDetailPage.jsx # 日记详情（查看、评论、评分、压缩率）
        ├── DiaryEditPage.jsx   # 日记编辑（创建/编辑、标签管理）
        ├── FoodPage.jsx        # 美食推荐（菜系筛选、距离排序、模糊搜索）
        ├── LoginPage.jsx       # 登录页
        ├── RegisterPage.jsx    # 注册页（含兴趣标签选择）
        ├── ProfilePage.jsx     # 个人中心（查看/编辑用户信息）
        └── IndoorNavPage.jsx   # 室内导航（楼层平面图、路径规划）
```

## 三、核心模块说明

### 3.1 路由配置 (`App.jsx`)

| 路由路径 | 组件 | 说明 |
|---------|------|------|
| `/` | HomePage | 首页 |
| `/scenic` | ScenicListPage | 景区/校园列表 |
| `/scenic/:id` | ScenicDetailPage | 景区详情 |
| `/route/:scenicId` | RoutePlanPage | 路线规划 |
| `/facility/:scenicId` | FacilityPage | 场所查询 |
| `/indoor/:buildingId` | IndoorNavPage | 室内导航 |
| `/diary` | DiaryListPage | 日记列表 |
| `/diary/:id` | DiaryDetailPage | 日记详情 |
| `/diary/edit/:id?` | DiaryEditPage | 日记编辑 |
| `/food` | FoodPage | 美食推荐 |
| `/login` | LoginPage | 登录 |
| `/register` | RegisterPage | 注册 |
| `/profile` | ProfilePage | 个人中心 |

### 3.2 API封装 (`services/api.js`)

使用Axios创建统一实例，配置：
- **baseURL**: `/api`（通过Vite代理转发到后端5000端口）
- **超时**: 30秒
- **凭证**: `withCredentials: true`（支持Cookie会话）
- **响应拦截器**: 统一错误处理、401自动提示登录

API模块划分：
- `authAPI` - 用户认证（登录、注册、登出、个人信息）
- `scenicAPI` - 景区/校园（列表、推荐、搜索、详情）
- `routeAPI` - 路线规划（最短路径、多点路线、地图数据）
- `facilityAPI` - 场所查询（附近设施、搜索、分类）
- `diaryAPI` - 旅游日记（CRUD、搜索、全文搜索、评论）
- `foodAPI` - 美食推荐（推荐、搜索、菜系、餐厅详情）
- `navAPI` - 室内导航（路径、建筑信息）

### 3.3 地图渲染 (`RoutePlanPage.jsx`, `IndoorNavPage.jsx`)

使用HTML5 Canvas原生绘制：
- **节点绘制**: 圆形标记，不同颜色区分起点/终点/途经点
- **边绘制**: 连线表示道路
- **路径高亮**: 虚线动画显示规划路径
- **交互**: 点击Canvas选择起点/终点/途经点
- **室内导航**: 按楼层偏移绘制，区分电梯/楼梯/走廊/房间

### 3.4 用户状态管理 (`App.jsx`)

使用React Context (`UserContext`) 管理全局用户状态：
- `user` - 当前登录用户信息
- `setUser` - 更新用户信息
- `loading` - 登录状态检查中

## 四、环境配置与启动

### 4.1 安装依赖
```bash
cd frontend
npm install
```

### 4.2 开发环境启动
```bash
npm run dev
```
默认运行在 `http://localhost:3000`，API请求自动代理到 `http://localhost:5000`。

### 4.3 生产构建
```bash
npm run build
```
构建产物在 `dist/` 目录。

## 五、关键设计决策

1. **Canvas vs 第三方地图**: 选择Canvas自绘地图，可完整展示自定义道路图数据和路径高亮，不依赖外部地图SDK
2. **Ant Design**: 提供完整的中文组件库，减少UI开发工作量
3. **TailwindCSS**: 与Ant Design配合使用（关闭preflight避免样式冲突），快速实现自定义布局
4. **Vite代理**: 开发环境下通过代理解决跨域问题，生产环境可通过Nginx反向代理
