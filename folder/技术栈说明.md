## 三、自实现核心算法与数据结构

> 以下算法和数据结构均为**自己设计并编程实现**，非调用第三方库。

### 3.1 数据结构

| 数据结构 | 文件位置 | 用途 |
|----------|----------|------|
| **Graph（邻接表图）** | `backend/app/services/graph_service.py` | 道路图存储，支持有向/无向加权图 |
| **MinHeap（最小堆）** | `backend/app/services/graph_service.py` | Dijkstra算法的优先队列 |
| **MaxHeap（最大堆）** | `backend/app/services/sort_service.py` | 排序辅助 |
| **MinHeapForTopK** | `backend/app/services/sort_service.py` | Top-K排序核心数据结构 |
| **SortableItem** | `backend/app/services/sort_service.py` | 可排序项封装 |
| **Trie（字典树）** | `backend/app/services/search_service.py` | 高效前缀匹配查找 |

### 3.2 核心算法

| 算法 | 文件位置 | 对应功能需求 |
|------|----------|------------|
| **Dijkstra最短路径** | `graph_service.py` | 路线规划 - 单点最短路径 |
| **多点TSP近似（贪心+2-opt）** | `graph_service.py` | 路线规划 - 途经多点最优路线 |
| **Top-K排序（最小堆）** | `sort_service.py` | 旅游推荐/日记推荐/美食推荐 - 不完全排序 |
| **多维度加权评分** | `sort_service.py` | 综合推荐（热度+评价+兴趣匹配） |
| **编辑距离（Levenshtein）** | `search_service.py` | 模糊查找算法 |
| **Trie前缀查找** | `search_service.py` | 高效名称查找 |
| **BM25F全文搜索** | `search_service.py` | 日记全文检索（基于Whoosh） |
| **zlib无损压缩** | `diary.py` (模型层) | 日记内容压缩存储 |
| **路径距离排序** | `sort_service.py` | 场所查询 - 按实际可达路径距离排序 |

## 四、系统架构

```
前端 (React + Vite)  <--HTTP/JSON-->  后端 (Flask API)  <--ORM-->  SQLite数据库
         |                                   |
    Ant Design UI                    核心算法服务层
    TailwindCSS                     (图算法/排序/搜索)
    Canvas地图绘制                    Whoosh全文索引
```

## 五、技术选型理由

1. **Flask**: 轻量级，学习成本低，适合课程设计项目，扩展性好
2. **SQLite**: 无需安装数据库服务器，单文件存储，便于部署和演示
3. **React + Vite**: 现代前端开发体验，热更新快，组件化开发
4. **Ant Design**: 成熟的企业级UI组件库，开箱即用，中文友好
5. **Whoosh**: 纯Python实现，无需Java/Elasticsearch等外部依赖
6. **Canvas**: 原生HTML5画布，用于地图和路径的图形化展示
