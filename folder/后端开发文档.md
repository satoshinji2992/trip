# 个性化旅游系统 - 后端开发文档

## 一、项目概述

后端采用 **Python 3.9+ / Flask 3.0** 框架构建RESTful API，使用SQLite数据库，核心算法（图算法、排序算法、查找算法）均基于自设计的数据结构自行编程实现。

## 三、数据模型设计

### 3.1 ER关系

```
User (1) ---> (N) Diary
Scenic (1) ---> (N) Building
Scenic (1) ---> (N) Facility
Scenic (1) ---> (N) GraphNode
Scenic (1) ---> (N) GraphEdge
Scenic (1) ---> (N) Restaurant
Building (1) ---> (N) IndoorNode
Building (1) ---> (N) IndoorEdge
Restaurant (1) ---> (N) Food
Diary (1) ---> (N) DiaryComment
Diary (1) ---> (N) DiaryImage
```

### 3.2 核心表结构

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `users` | 用户表（12+） | id, username, password_hash, interests(JSON) |
| `scenics` | 景区/校园表（200+） | id, name, type, category, popularity, rating, tags(JSON) |
| `buildings` | 建筑物表（20+） | id, scenic_id, name, type, floors, has_elevator |
| `facilities` | 服务设施表（50+） | id, scenic_id, name, type, graph_node_id, rating |
| `graph_nodes` | 道路图节点 | id, scenic_id, name, x, y, latitude, longitude |
| `graph_edges` | 道路图边（200+） | id, scenic_id, from/to_node_id, distance, congestion, transport_allowed |
| `restaurants` | 餐厅表 | id, scenic_id, name, cuisine, graph_node_id, rating, popularity |
| `foods` | 美食表 | id, restaurant_id, name, cuisine, price, rating |
| `diaries` | 日记表 | id, user_id, title, content_compressed(BLOB), view_count, rating_sum |
| `diary_comments` | 日记评论表 | id, diary_id, user_id, content, rating |
| `diary_images` | 日记图片表 | id, diary_id, image_path |
| `indoor_nodes` | 室内导航节点 | id, building_id, name, node_type, floor, x, y |
| `indoor_edges` | 室内导航边 | id, building_id, from/to_node_id, distance, edge_type |

## 四、核心算法详解

### 4.1 Dijkstra最短路径算法 (`graph_service.py`)

- **数据结构**: 自实现MinHeap（最小堆）作为优先队列
- **时间复杂度**: O((V+E)logV)
- **支持**: 最短距离和最短时间两种权重
- **交通工具适配**: 根据交通模式过滤可通行的边，调整速度

### 4.2 多点最短路径TSP近似 (`graph_service.py`)

- **步骤1**: 计算所有关键点之间的最短距离矩阵（调用Dijkstra）
- **步骤2**: 贪心算法构建初始解（最近邻法）
- **步骤3**: 2-opt局部优化（反转子路径减少总距离）
- **支持**: 是否返回起点选项

### 4.3 Top-K排序算法 (`sort_service.py`)

- **数据结构**: 自实现MinHeapForTopK（容量为K的最小堆）
- **时间复杂度**: O(N·logK)，优于完全排序的O(N·logN)
- **应用**: 景区推荐、日记推荐、美食推荐——只需排好前10个

### 4.4 推荐评分算法 (`sort_service.py`)

综合评分公式：`score = 0.4 × 热度归一化 + 0.3 × 评价归一化 + 0.3 × 兴趣匹配度`

### 4.5 模糊查找算法 (`search_service.py`)

- **编辑距离**: 自实现Levenshtein距离DP算法
- **匹配策略**: 完全包含 > 分词匹配 > 子串模糊匹配
- **Trie树**: 自实现字典树，用于高效前缀查找

### 4.6 全文搜索 (`search_service.py`)

- 基于Whoosh库实现BM25F评分
- 支持标题、内容、目的地、标签多字段检索

### 4.7 无损压缩存储 (`diary.py`)

- 使用zlib level=9压缩日记内容
- `set_content()` 压缩存储，`get_content()` 解压读取
- API返回压缩率信息

### 4.8 路径距离排序 (`sort_service.py`)

- 一次Dijkstra计算从源点到所有节点的最短距离
- 按**实际可达路径距离**排序（非直线距离）

## 五、环境配置与启动

### 5.1 安装依赖
```bash
cd backend
pip install -r requirements.txt
```

### 5.2 初始化数据
```bash
python init_data.py
```

### 5.3 启动服务
```bash
python run.py
```
默认运行在 `http://localhost:5001`。

### 5.4 Gunicorn + Nginx 多 Worker 部署

开发环境使用 Flask 内置服务即可；生产/答辩讲解并发支持时，可以使用 Nginx 反向代理到 Gunicorn 多 worker：

```bash
cd backend
gunicorn -c ../deploy/gunicorn.conf.py run:app
```

Nginx 配置文件位于：

```text
deploy/nginx.trip.conf
```

架构：

```text
浏览器 -> Nginx 多 worker -> Gunicorn 多 worker -> Flask API
```

Nginx 负责静态资源、反向代理和并发连接；Gunicorn 负责启动多个 Flask worker 处理后端接口。

## 六、拥挤度和交通工具策略

### 6.1 拥挤度策略
- 每条道路边有 `congestion` 属性（0~1）
- 实际速度 = max(0.1, 1-congestion) × 基础速度
- 最短时间策略会自动考虑拥挤度
- 初始化脚本会写入确定性的拥挤度分布：主路和热门短连线更拥挤，支路更通畅
- 因此同一组起终点下，“最短距离”可能选择近但堵的路，“最短时间”可能选择绕行但畅通的路

### 6.2 交通工具策略
- **校区内**: 步行(5km/h) + 自行车(15km/h)，自行车任何地点都有
- **景区内**: 步行(5km/h) + 电瓶车(20km/h)，电瓶车路线固定
- 每条边有 `transport_allowed` 属性限制可通行的交通工具
