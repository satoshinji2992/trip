# 个性化旅游系统 - 用户使用手册

## 一、系统简介

个性化旅游系统是一个覆盖旅游前、旅游中、旅游后全流程的Web应用，提供旅游地点推荐、路线规划、场所查询、旅游日记交流、美食推荐、室内导航等六大核心功能。系统突出微调并本地部署的Wan2.2图生视频大模型、Nginx + Gunicorn一键部署、日记压缩存储、全文检索和道路拥挤度路线规划等创新点。所有核心算法（图算法、排序算法、查找算法）均基于自设计数据结构、自行编程实现。

## 二、创新点一览

| 创新点 | 实现方式 | 用户/运行价值 |
| --- | --- | --- |
| 微调并本地部署Wan2.2大模型 | 通过本地Wan2.2图生视频服务生成旅游动画 | 游记可从静态图片扩展为短视频，展示效果更强 |
| Nginx + Gunicorn一键部署 | `setup_and_start_nginx.sh` 自动安装依赖、初始化数据、构建前端并启动服务 | 可执行包更容易复现运行环境 |
| zlib压缩存储 | 日记正文压缩后存入SQLite | 降低存储占用，保持内容无损恢复 |
| BM25F全文检索 | Whoosh索引标题、正文、目的地、标签 | 支持按正文内容搜索旅游日记 |
| 道路拥挤度路线规划 | 道路边记录拥挤度，最短时间按速度修正计算 | 最短距离和最短时间可产生不同路线，更贴近实际 |
| 实际路径距离排序 | 场所查询使用Dijkstra计算道路距离 | 附近设施排序更真实，不依赖直线距离 |

## 三、功能一览

| 功能模块 | 核心能力 | 核心算法 |
|----------|---------|---------|
| 🧭 旅游推荐 | 景区/校园浏览、搜索、个性化推荐排序 | Top-K排序（最小堆）、Trie前缀匹配、模糊查找（编辑距离） |
| 🗺️ 路线规划 | 单点最短路径、多点旅游线路、交通工具/拥挤度策略 | Dijkstra最短路径、途经多点TSP（贪心+2-opt） |
| 🏠 室内导航 | 建筑内部楼层导航（大门→电梯→房间） | Dijkstra（室内图） |
| 🔍 场所查询 | 附近设施查找、按实际路径距离排序 | Dijkstra + 路径距离排序 |
| 📖 旅游日记 | 撰写/浏览/评分/全文搜索/压缩存储、图片上传、AIGC旅游动画 | BM25F全文搜索、zlib无损压缩、Top-K推荐排序、Wan2.2图生视频接口 |
| 🍜 美食推荐 | 菜系筛选、模糊搜索、热度/评价/距离排序 | 模糊查找（编辑距离）、Top-K排序 |

## 五、技术栈

- **后端**: Python 3.9+ / Flask 3.0 / SQLAlchemy / SQLite / Whoosh / Gunicorn
- **前端**: React 18 / Vite 5 / Ant Design 5 / TailwindCSS 3 / Axios
- **AIGC服务**: 微调并本地部署的Wan2.2图生视频服务，支持图片、提示词、分辨率、帧数、采样步数和seed参数
- **部署**: Nginx多worker反向代理 + Gunicorn多worker后端服务，发布包支持一键安装、初始化和运行
- **自实现数据结构**: 邻接表图、最小堆、最大堆、Top-K堆、Trie字典树、可排序项
- **自实现算法**: Dijkstra、TSP近似、Top-K排序、编辑距离、模糊匹配、路径距离排序

## 六、快速启动

### 6.1 环境要求

- Python 3.9+
- Node.js 18+
- npm 9+

### 6.2 后端启动

```bash
# 1. 进入后端目录
cd backend

# 2. 安装Python依赖
pip install -r requirements.txt

# 3. 如需重建数据，可初始化数据库；发布包已自带可演示 trip.db
python init_data.py

# 4. 启动后端服务
python run.py
```

后端运行在 `http://localhost:5001`

### 6.3 前端启动

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装Node依赖
npm install

# 3. 启动开发服务器
npm run dev
```

前端运行在 `http://localhost:3000`，API请求自动代理到后端。

## 七、功能演示指南

### 6.1 旅游推荐演示
1. 打开首页，查看智能推荐的景区列表
2. 点击「旅游推荐」，切换排序方式（综合/热度/评价）
3. 搜索"故宫"，查看模糊匹配结果
4. 登录后，系统会根据用户兴趣标签个性化排序

### 6.2 路线规划演示
1. 进入任意景区详情页，点击「路线规划」
2. 在地图上点击选择起点（绿色）和终点（红色）
3. 选择策略（最短距离/最短时间）和交通工具
4. 点击「开始规划」，查看路径高亮、平均拥挤度和路段拥挤情况
5. 切换到「多点规划」模式，添加多个途经点，查看TSP优化的游览顺序

### 6.3 室内导航演示
1. 进入景区详情页，点击某个多层建筑物
2. 在室内导航页选择起点（如1层入口）和终点（如3层302室）
3. 查看导航路径和分步指令（含电梯/楼梯切换）

### 6.4 场所查询演示
1. 进入景区详情页，点击「场所查询」
2. 选择当前位置节点
3. 点击「查找附近设施」，查看按路径距离排序的结果
4. 尝试按类别过滤（如洗手间、超市）
5. 搜索"咖啡"，查看模糊匹配结果

### 6.5 旅游日记演示
1. 点击「旅游日记」浏览所有公开日记
2. 切换排序方式（综合/热度/评价）
3. 使用普通搜索和全文搜索
4. 进入日记详情，查看压缩率信息
5. 登录后发表评论和评分
6. 点击「写日记」创建新日记并上传图片
7. 在日记详情中选择图片、修改提示词和视频参数，调用Wan服务生成旅游动画

### 6.6 美食推荐演示
1. 点击「美食推荐」，选择一个景区/校园
2. 切换排序方式和菜系过滤
3. 选择当前位置后按距离排序
4. 搜索"红烧肉"，查看模糊匹配的菜品和餐厅

## 九、核心算法说明

### 9.1 自实现数据结构（非调用第三方库）

| 数据结构 | 位置 | 用途 |
|----------|------|------|
| Graph（邻接表） | `services/graph_service.py` | 存储道路图/室内图 |
| MinHeap（最小堆） | `services/graph_service.py` | Dijkstra优先队列 |
| MaxHeap（最大堆） | `services/sort_service.py` | 排序辅助 |
| MinHeapForTopK | `services/sort_service.py` | Top-K排序，容量K的最小堆 |
| Trie（字典树） | `services/search_service.py` | 高效前缀匹配 |
| SortableItem | `services/sort_service.py` | 可排序项封装 |

### 9.2 自实现算法

| 算法 | 复杂度 | 对应需求 |
|------|--------|----------|
| Dijkstra最短路径 | O((V+E)logV) | 路线规划、场所距离排序 |
| TSP近似（贪心+2-opt） | O(N²·I) | 多点旅游路线 |
| Top-K排序 | O(NlogK) | 推荐排序（不完全排序） |
| 编辑距离（Levenshtein） | O(M·N) | 模糊查找 |
| Trie前缀查找 | O(L) | 高效名称查找 |
| zlib无损压缩 | - | 日记压缩存储 |
| BM25F全文搜索 | - | 日记内容检索 |
| 多维度加权评分 | O(1) | 综合推荐 |

## 十、注意事项

1. **首次使用**必须先运行 `python init_data.py` 初始化数据库
2. 前后端需要**同时启动**才能正常使用
3. 前端开发服务器默认代理API到5001端口，确保后端已启动
4. SQLite数据库文件为 `backend/trip.db`，删除后可重新初始化
5. 全文搜索索引目录为 `backend/whoosh_index/`
6. 上传文件和生成视频存储在 `backend/uploads/`
7. 视频生成服务器地址通过 `backend/.env` 中的 `AIGC_BASE_URL` 和 `AIGC_ENDPOINT` 配置；没有 `.env` 时使用 `config.py` 中的默认值
8. 可运行发布包可通过 `./scripts/build_release.sh` 生成，输出到 `release/trip-runnable.zip`
